CNN-LSTM Multi-Task Service

Hybrid Deep Learning - Kombiniert CNN Feature-Extraktion mit LSTM Sequenzverarbeitung

🧠 Beschreibung

Der CNN-LSTM Multi-Task Service kombiniert Convolutional Neural Networks (CNN) für lokale Feature-Extraktion mit Long Short-Term Memory (LSTM) Netzwerken für sequenzielle Musterverarbeitung. Diese Hybrid-Architektur ermöglicht die gleichzeitige Lösung von drei Aufgaben in einem einzigen Modell.

Das Multi-Task Learning teilt gemeinsame Repräsentationen zwischen verwandten Aufgaben, was zu besserer Generalisierung und effizienterer Nutzung der Trainingsdaten führt. Der Service verarbeitet 25 technische Features über variable Sequenzlängen je nach Timeframe.

🎯 Multi-Task Vorhersagen

Der Service löst drei zusammenhängende Aufgaben gleichzeitig:

Preis-Vorhersage Regression

Vorhersage der erwarteten Preisänderung für 4 verschiedene Zeithorizonte relativ zum aktuellen Preis.

Outputs
  • 1h Forecast
  • 4h Forecast
  • 1d Forecast
  • Direction & Confidence
Pattern-Klassifikation Multi-Label

Erkennung von 16 klassischen Chart-Patterns mit Wahrscheinlichkeiten für jedes Pattern (Multi-Label).

16 Patterns
  • Head & Shoulders
  • Double/Triple Top/Bottom
  • Triangles, Wedges, Flags
  • Cup & Handle, Rectangle
Regime-Vorhersage 4-Klassen

Klassifikation des aktuellen Marktregimes in eine von vier Kategorien mit Wahrscheinlichkeiten.

Regime-Typen
  • Bull Trend
  • Bear Trend
  • Sideways/Range
  • High Volatility

📦 Modell-Architektur

Die Hybrid CNN-LSTM Architektur besteht aus drei Hauptkomponenten:

Input: (batch, seq_len, 25 features)
              │
              ▼
┌─────────────────────────────────────┐
│         CNN ENCODER                 │
│  ┌─────────────────────────────┐    │
│  │ Conv1d(25→64, k=3) + BN     │    │
│  │ ReLU + Dropout(0.2)         │    │
│  ├─────────────────────────────┤    │
│  │ Conv1d(64→128, k=3) + BN    │    │
│  │ ReLU + Dropout(0.2)         │    │
│  ├─────────────────────────────┤    │
│  │ Conv1d(128→256, k=3) + BN   │    │
│  │ ReLU + Dropout(0.2)         │    │
│  └─────────────────────────────┘    │
└─────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────┐
│         LSTM ENCODER                │
│  ┌─────────────────────────────┐    │
│  │ BiLSTM(256→128, layers=2)   │    │
│  │ Dropout(0.3)                │    │
│  ├─────────────────────────────┤    │
│  │ Attention Mechanism         │    │
│  │ Output: 256-dim vector      │    │
│  └─────────────────────────────┘    │
└─────────────────────────────────────┘
              │
    ┌─────────┼─────────┐
    ▼         ▼         ▼
┌───────┐ ┌───────┐ ┌───────┐
│ Price │ │Pattern│ │Regime │
│ Head  │ │ Head  │ │ Head  │
│ (4)   │ │ (16)  │ │ (4)   │
└───────┘ └───────┘ └───────┘
Multi-Task Loss: Total = 0.4×Price_MSE + 0.35×Pattern_BCE + 0.25×Regime_CE
Die Gewichtung balanciert die verschiedenen Aufgaben für optimales gemeinsames Lernen.

⚙️ Funktionsweise

Der CNN-LSTM Algorithmus funktioniert durch:

  • CNN Feature-Extraktion: Erkennt lokale Muster wie Kerzenformationen und kurzfristige Trends
  • Batch Normalization: Stabilisiert das Training und beschleunigt die Konvergenz
  • Bidirectionales LSTM: Verarbeitet Sequenzen vorwärts und rückwärts für vollständigen Kontext
  • Attention Mechanism: Fokussiert auf die wichtigsten Zeitpunkte in der Sequenz
  • Multi-Task Heads: Spezialisierte Ausgabeschichten für jede Aufgabe
CPU-Optimiert: Der Service ist für CPU-Inferenz optimiert mit Thread-Limiting (OMP_NUM_THREADS=2). GPU-Support wird mit PyTorch 2.6+ für NVIDIA Thor (sm_110) verfügbar.
