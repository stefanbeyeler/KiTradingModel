Redis Cache

Zentraler In-Memory Cache für alle Microservices mit LRU-Eviction und Persistenz

📊 Beschreibung

Redis ist der zentrale Cache des KI Trading Model Systems. Er speichert Marktdaten, API-Antworten und berechnete Indikatoren zwischen, um die Anzahl der externen API-Aufrufe zu minimieren und die Antwortzeiten zu optimieren.

Alle Services (Data, NHITS, TCN, HMM, RAG, LLM) nutzen Redis als gemeinsamen Cache. Der CacheService in src/services/cache_service.py bietet eine einheitliche Schnittstelle mit kategoriebasiertem TTL-Management.

Redis 7 Alpine Linux 512 MB RAM LRU Eviction AOF Persistenz

⚙️ Funktionen

🚀 In-Memory Speed

Mikrosekunden-Zugriffszeiten durch RAM-basierte Datenhaltung.

📋 LRU-Eviction

Automatisches Entfernen der am längsten ungenutzten Einträge bei Speicherknappheit.

💾 AOF Persistenz

Append-Only-File für Datenwiederherstellung nach Container-Neustart.

📈 TTL-Management

Kategoriebasierte Time-To-Live Werte für automatische Cache-Invalidierung.

Fallback: Wenn Redis nicht erreichbar ist, fällt der CacheService automatisch auf einen In-Memory Cache zurück. Das System bleibt funktionsfähig, verliert aber den verteilten Cache.

🕑 Cache-Kategorien & TTL

Jede Datenkategorie hat eine eigene Time-To-Live (TTL) Konfiguration:

Kategorie TTL Beschreibung
MARKET_DATA 60s Echtzeit-Kurse, schnelle Updates
OHLCV 5 min Kerzendaten (Open, High, Low, Close, Volume)
INDICATORS 5 min Technische Indikatoren (RSI, MACD, etc.)
SYMBOLS 1 h Symbol-Listen und Metadaten
SENTIMENT 15 min Fear & Greed, Social Media Sentiment
ECONOMIC 30 min Wirtschaftskalender-Events
ONCHAIN 10 min On-Chain Daten (Whale Alerts, Exchange Flows)
TRAINING 6 h Training-Daten für ML-Modelle

🔧 Konfiguration

Redis-Konfiguration im Docker-Compose:

Parameter Wert Beschreibung
maxmemory 512 MB Maximaler Speicherverbrauch
maxmemory-policy allkeys-lru Eviction-Strategie: Least Recently Used
appendonly yes AOF-Persistenz aktiviert
port 6379 Standard Redis-Port
Hinweis: Redis ist nur intern im Docker-Netzwerk erreichbar. Externer Zugriff ist aus Sicherheitsgründen nicht konfiguriert.

💻 Verwendung im Code

Beispiel für die Cache-Nutzung in Python:

from src.services.cache_service import cache_service, CacheCategory

# Daten aus Cache holen
cached = await cache_service.get(
    CacheCategory.OHLCV,
    symbol,
    params={"limit": 500}
)

# Daten in Cache speichern
await cache_service.set(
    CacheCategory.OHLCV,
    data,
    symbol,
    params={"limit": 500}
)

# Cache-Kategorie löschen
await cache_service.clear_category(CacheCategory.MARKET_DATA)

# Cache-Statistiken
stats = cache_service.get_stats()
