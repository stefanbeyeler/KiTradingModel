Trading Workplace

Vollautomatische Trading-Setup-Aggregation mit Multi-Signal-Scoring

Übersicht

Der Trading Workplace Service aggregiert Vorhersagen aller ML-Services und berechnet einen gewichteten Composite-Score für Trading-Setups. Dem Trader werden die vielversprechendsten Setups automatisch präsentiert.

Schnellbeurteilung

Top Trading-Setups auf einen Blick mit Composite-Score

Multi-Signal-Scoring

Gewichtete Kombination aus NHITS, HMM, TCN, Candlestick

Vertiefte Analyse

RAG-Kontext + LLM-generierte Empfehlungen

Konfigurierbare Watchlist

Eigene Symbole mit Alert-Schwellen

FastAPI Python 3.11 Async httpx Pydantic

Signal-Gewichte

Der Composite-Score wird aus den gewichteten Signalen aller ML-Services berechnet:

Service Gewicht Signal
NHITS 30% trend_probability, direction
HMM 25% regime_confidence, alignment
TCN 20% pattern_confidence
Candlestick 15% pattern_strength
Technical 10% trend_alignment (RSI, MACD)
Alignment-Bonus: Wenn alle Signale in die gleiche Richtung zeigen, wird ein Bonus von 10% auf den Score angewendet. Bei widersprüchlichen Signalen erfolgt eine Reduktion um 10%.

API Endpoints

Alle Endpoints sind unter /workplace/api/v1/ erreichbar.

Setups

GET /setups/ Top Trading-Setups (Schnellbeurteilung)
GET /setups/{symbol} Setup für einzelnes Symbol
POST /setups/{symbol}/refresh Setup aktualisieren

Analyse

POST /analyze/{symbol} Vertiefte Analyse mit RAG + LLM
POST /analyze/{symbol}/quick Schnelle Analyse (ohne LLM)

Watchlist

GET /watchlist/ Watchlist abrufen
POST /watchlist/ Symbol hinzufügen
DELETE /watchlist/{symbol} Symbol entfernen

Scanner

GET /scan/status Scanner-Status
POST /scan/start Scanner starten
POST /scan/stop Scanner stoppen
POST /scan/trigger Manuellen Scan auslösen

Konfiguration

Wichtige Umgebungsvariablen:

WORKPLACE_SCAN_INTERVAL_SECONDS Scan-Intervall (Standard: 60)
WORKPLACE_AUTO_SCAN_ENABLED Auto-Scan aktiviert (Standard: true)
WORKPLACE_HIGH_CONFIDENCE_THRESHOLD High-Confidence ab Score (Standard: 75)
Hinweis: Die Signal-Gewichte können über Umgebungsvariablen angepasst werden (z.B. WORKPLACE_NHITS_WEIGHT=0.35). Die Summe aller Gewichte muss 1.0 ergeben.
