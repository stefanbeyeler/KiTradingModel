👁 Watchdog Service

Microservice-Monitoring, Testing und Telegram-Alerts

📊 Beschreibung

Der Watchdog Service ist das zentrale Monitoring- und Testing-System des KI Trading Model. Er überwacht kontinuierlich alle Microservices auf Verfügbarkeit und Gesundheit, führt automatisierte API-Tests durch und sendet bei Problemen sofortige Benachrichtigungen über Telegram.

Der Service führt regelmäßig Health-Checks durch, erkennt Ausfälle und Recovery-Events, bietet ein umfangreiches Testing-Framework mit ~90 Tests und implementiert ein intelligentes Alert-System mit Cooldown-Mechanismus.

⚙️ Funktionsweise

Der Service bietet folgende Hauptfunktionen:

🔴 Health Monitoring

Regelmäßige Health-Checks aller Services über deren /health Endpoints mit konfigurierbarem Intervall.

📩 Telegram Alerts

Sofortige Benachrichtigungen bei Service-Ausfällen und Recovery-Events an konfigurierte Chat-IDs.

⏰ Alert Cooldown

Intelligenter Cooldown-Mechanismus verhindert wiederholte Alerts für denselben Service.

📈 Status Dashboard

API-Endpoints für Echtzeit-Status aller überwachten Services und Alert-Historie.

🎯 Criticality Levels

Services werden nach Kritikalität (critical, high, medium, low) klassifiziert für priorisierte Alerts.

🔄 Auto-Recovery Detection

Automatische Erkennung wenn ein ausgefallener Service wieder verfügbar wird.

🧪 Integriertes Testing

~90 API-Tests für alle Services mit Smoke, API, Contract und Integration Tests.

📺 Live Test-Streaming

Echtzeit-Updates via Server-Sent Events (SSE) während Test-Runs.

📊 Service Runs History

Detaillierte Historie aller Validierungen, Trainings und Scans über alle Services hinweg.

Architektur: Der Watchdog lauft als eigenstandiger FastAPI-Service im Docker-Netzwerk und kann alle anderen Container uber deren Service-Namen erreichen. Er ist bewusst lightweight gehalten, um selbst minimale Ressourcen zu verbrauchen.

🧪 Testing-System

Der Watchdog enthält ein vollständiges Testing-Framework für alle Microservices. Tests können manuell oder automatisiert ausgeführt werden.

💥 Smoke Tests

Schnelle Health-Checks (< 1s/Test) für alle Services.

🔗 API Tests

Vollständige Endpoint-Tests mit Request/Response-Validierung.

📜 Contract Tests

Schema-Validierung der API-Responses.

🔁 Integration Tests

Service-übergreifende Workflows testen.

Makefile-Integration: Tests konnen einfach via make test-smoke, make test-full oder make test-service SERVICE=data ausgefuhrt werden.

Test-Prioritaten

Prioritat Beschreibung Beispiele
Critical Grundlegende Funktionen Health-Checks, Data Gateway
High Wichtige Features OHLCV-Daten, Forecasts
Medium Standard-Funktionen Indicators, Patterns
Low Optionale Features Bulk-Operations

📩 Telegram Integration

Der Watchdog verwendet die Telegram Bot API für Benachrichtigungen. Alerts werden mit Emoji-Icons und formatiertem Text gesendet für optimale Lesbarkeit auf mobilen Geräten.

🔴 Service Down

Sofortige Benachrichtigung wenn ein Service nicht erreichbar ist.

🟢 Service Recovery

Bestätigung wenn ein ausgefallener Service wieder online ist.

Telegram Setup: Um Alerts zu erhalten, erstellen Sie einen Bot über @BotFather und konfigurieren Sie TELEGRAM_BOT_TOKEN und TELEGRAM_CHAT_IDS in der .env.watchdog Datei.

🔧 Konfiguration

Der Watchdog wird über Umgebungsvariablen konfiguriert:

Variable Beschreibung Default
WATCHDOG_PORT Port des Watchdog-Service 3010
CHECK_INTERVAL_SECONDS Intervall zwischen Health-Checks 30
ALERT_COOLDOWN_MINUTES Mindestzeit zwischen Alerts für denselben Service 15
TELEGRAM_ENABLED Telegram-Benachrichtigungen aktivieren true
TELEGRAM_BOT_TOKEN Bot-Token von @BotFather -
TELEGRAM_CHAT_IDS Komma-separierte Liste von Chat-IDs -
